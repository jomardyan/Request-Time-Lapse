üîç Filtering Requests

Status Code Filter

Enter HTTP status codes (comma-separated): 200,201,204

Common status codes:

  • 2xx (200, 201, 204) - Success responses
  • 3xx (301, 302, 304) - Redirects
  • 4xx (400, 401, 403, 404) - Client errors
  • 5xx (500, 502, 503) - Server errors

Latency Filter

Enter maximum latency in milliseconds: 1000

Shows only requests that completed within the specified time.

Response Content Filter

Search in response body: error, success, etc.

Case-insensitive search through response content.

üí° Tip: Combine multiple filters to narrow down results quickly!

Export Formats

JSON Export: Full request data with all details - best for data processing
CSV Export: Spreadsheet format - easy to analyze in Excel/Sheets
HTML Export: Beautiful HTML report - great for sharing with team

How to Export

  1. Click the export button or press Ctrl + E
  2. Choose your desired format
  3. The file will download automatically
  4. Open in your preferred application
üí° Tip: Use CSV format to create pivot tables and charts in Excel!

üîÑ Comparing Snapshots

How to Use Compare Mode

Step 1: Press Ctrl + Shift + C to enable compare mode
Step 2: Click on the first request snapshot to select it
Step 3: Click on the second request snapshot to compare
Step 4: Review the side-by-side comparison with highlighted differences
üí° Use Case: Compare API responses before and after a code change to ensure regression didn't occur.

üíæ Storage & Performance

Storage Indicator

Shows your current storage usage. The indicator changes color:

  • Green: Under 50% - plenty of space
  • Yellow: 50-89% - getting full
  • Red: 90%+ - warning, clear old data

Clearing Data

To free up space, clear old requests from the panel or export them first.

‚ö†Ô∏è Warning: Clearing data is permanent. Export before clearing if you need to preserve the data!

üêõ Troubleshooting

Requests not appearing?

1. Ensure DevTools is open and the Request Time-Lapse tab is active
2. Try refreshing the page (F5)
3. Make sure storage permissions are enabled

Storage full warning?

1. Export your current data to preserve it
2. Clear the storage from the indicator
3. Reload the page to start fresh

Performance issues?

1. Filter requests to show only what you need
2. Clear old data periodically
3. Restart DevTools if it becomes sluggish
